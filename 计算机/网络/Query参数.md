# Query参数
### 定义
Query参数（也常称为`URL`参数、查询参数），是直接拼接在`URL`末尾、以键值对形式传递的参数集合，是`HTTP`的`GET`请求中传递数据的核心方式

### 特性

- `Query`参数是键值对结构，键和值均为**字符串**类型
- 通过`?`与`URL`主路径分隔，多个参数之间用`&`连接，键和值用`=`绑定
- 参数拼接在`URL`的**查询字符串**部分，格式为：`http://域名/路径?key1=value1&key2=value2`

# 优缺点
### 优点

- 传递简单：无需复杂的请求体构造，直接拼接在`URL`中，前端实现成本低
- 兼容性强：所有`HTTP`客户端 / 服务端框架均原生支持解析`Query`参数，无跨平台适配成本

### 缺点

- 长度限制：不同浏览器或服务器对`URL`长度有上限，无法传递大量数据
- 安全性低：参数是**明文传递**的，不适合传递敏感数据（密码等）
- 数据类型受限：**仅支持字符串类型**，传递数组、对象等需先序列化为字符串，服务端需反序列化

# 示例
假设我要给`https://www.example.com/school`这个路径传递两个参数，一个参数的键是`name`，值为`xiwangxiaoxue`；另一个参数键是`class`，值为`2`

那么我们最终的`URL`就是这样：

`https://www.example.com/school?name=xiwangxiaoxue&class=2`

# 注意事项

- 空值处理：值为`null/undefined`的参数建议不拼接，避免传递key=null或key=undefined
- 编码一致性：前端用统一的`URL`编码方法处理参数，服务端需用对应解码方法还原
- 大小写敏感：`Query`参数的键名通常大小写敏感（如`Page`和`page`视为不同键）
- 特殊字符：键值出现特殊字符（中文、`&`、`=`等）需经过`URL`编码，避免破坏`URL`结构

### Unity的编码方法
可以使用`UnityWebRequest.EscapeURL()`对存在特殊字符的值进行编码
