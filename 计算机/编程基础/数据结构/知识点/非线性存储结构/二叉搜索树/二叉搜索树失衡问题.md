# 二叉树失衡问题
在前面的文章中，我们讲解了二叉搜索树的原理，其可以实现 $O(log_2 n)$级别时间复杂度的插入、删除以及查找操作，
但是这是较理想的情况，在部分情况下，二叉树搜索树会退化成 $O(n)$级别的时间复杂度

### 问题
我们来看一下下面这个场景，我们按`1 2 3 4 5`的顺序添加元素，最终我们得到的二叉搜索树是下面这样的
```plainText
1
 \
  2
   \
    3
     \
      4
       \
        5
```

此时，二叉搜索树已经退化成了一个链表，对其进行操作的时间复杂度已经变成了 $O(n)$

### 解决

为了解决“二叉树失衡”的问题，我们将在二叉树搜索树的基础上进行优化，便是**平衡二叉树**
