# 一、基本知识

### 定义	

​	数组由相同数据类型的元素组成，在计算机内存中以**连续地址空间**存储的线性集合，元素通过「索引（下标）」唯一标识位置。

### 特性

- 数组是一种**线性存储**的数据结构，其在内存中是**连续分配空间**的。
- 数组本身存储的是数组第一个元素的地址（**首地址**）。
- 我们常说的“数组索引”、“数组下标”，其实是“**数组中某个元素相对于第一个元素的偏移量**”。
- 这也就是为什么数组下标是从0开始的原因，因为第一个元素相对于其自身没有偏移。

# 二、优缺点

### 优点

- 随机访问效率高：通过索引直接定位元素，直接计算内存地址，时间复杂度 O(1)。
- 存储密度高：无额外开销，仅在内存中存储元素本身。

### 缺点

- 长度固定：初始化后长度不可修改。
- 内存浪费：若实际存储元素少于初始化长度，剩余空间会浪费。

# 三、关键操作

### 访问操作

​	访问操作，即读取第x个元素，对于数组而言，该操作的时间复杂度是 O(1)。

​	这是因为，**元素的索引是该元素相对于第一个元素的偏移量**（也就是偏移了多少个元素），由于数组的存储结构是线性的，并且数组的类型是确定的（即每一个元素占用的空间是已知的），我们可以通过如下的公式计算出索引为X的元素的地址：
$$
索引为X的元素的地址 = 数组的首地址 + 每一个元素占用的内存大小 * X
$$
​	这样，我们可以直接取该地址存储的数据便可得到该元素。

### 查找操作

​	查找操作，即给出一个目标值，查找该目标值是否在数组中。

​	数组的查找主要有两种方法：线性查找和二分查找。

### 插入、删除操作

​	对于静态数组而言，其创建后不可改变其长度，所以数组本质上是不可以进行插入和删除操作的。
